# 头脑风暴 (Brainstorm)

三人小组结构化讨论。匹配三个不同专业领域的角色，通过发散→碰撞→聚焦三轮对话，形成可执行方案。

## 流程

### 1. 启动

1. 分析主题领域
2. 匹配三个角色（必须分属不同专业领域，覆盖战略/执行/用户三个维度）
3. 用表格介绍角色（角色名 | 职责 | 视角），不要分段描述
4. 执行研究准备（见第2步）
5. 开始讨论

### 2. 研究准备

角色介绍后、讨论前，按下表判断是否搜索：

| 主题特征 | 搜索？ | 模式 |
|---------|--------|------|
| 涉及具体数据、市场、政策、法规 | 是 | 深度（3-5次：1次通用 + 每角色1次定向） |
| 涉及最新技术、产品、公司动态 | 是 | 快速（1-2次通用） |
| 通用策略、创意、方法论、成熟领域 | 否 | 跳过 |

注意：基于语义判断而非关键词匹配。"最新的社交APP创意"中的"最新"是修饰语，不需要搜索。

**搜索后输出背景简报**（不超过200字）：列出3-5条关键事实即可。无需搜索则跳过，直接开始讨论。

### 3. 讨论

三轮，每轮三个角色各发言一次（A→B→C），连续输出不暂停。

**各轮定位与字数：**

| 轮次 | 定位 | 每人约 | 要求 |
|------|------|--------|------|
| 第一轮 | 发散 | 150字 | 从本专业角度抛出核心观点 |
| 第二轮 | 碰撞 | 200字 | 必须对前一位角色的至少一个观点提出质疑或替代方案 |
| 第三轮 | 聚焦 | 100字 | 收敛共识，标注关键分歧 |

**发言格式：**
```
### 【角色名】
观点内容（聚焦1个核心观点，不铺开）
```

**硬性规则：**
- 三个角色必须分属不同专业领域，禁止视角重叠
- 第二轮每人必须包含至少一处明确质疑（"但是…""我不同意…""这里有个问题…"）
- 每次发言聚焦1个核心观点，不面面俱到
- 引用搜索数据时自然融入分析，不复述原文
- 遇到不确定的事实性内容，主动触发搜索验证，不猜测
- 事实性主张必须有据可依，不编造数据

**用户介入：** 用户发言优先级最高，随时可插话引导方向。介入后从下一个角色继续。

### 4. 决策点

三轮完成后，简要呈现：已形成的共识、待解决的分歧。

询问用户：
> 1. **继续讨论** - 再进行三轮深入
> 2. **总结方案** - 整合形成最终方案

### 5. 总结方案

不超过500字。根据讨论内容动态组织结构，不套模板。

必须包含：核心结论、可执行的行动建议、仍需决策的分歧点。
可选包含：实施步骤、资源需求、风险、成功指标。

## 注意事项

- 角色介绍用表格，一行一个角色
- 背景简报（如有）不超过200字
- 三轮连续输出，三轮后才暂停等用户决策
- 总结不超过500字，聚焦结论和行动项
- 用户可随时介入、换方向、重新开始
