# Phase 3: PLAN - Break Into Steps

## Overview

**Goal:** Transform approved design into specific, actionable tasks.

**When to Use:** After DESIGN phase is approved.

**Success Criteria:** Clear roadmap with verifiable tasks.

## The Process

### Step 1: Decompose into Bite-Sized Tasks

**Task Size Rule:** Each task should take 2-10 minutes.

**Why small tasks?**
- Easy to estimate accurately
- Clear progress tracking
- Low switching cost
- Obvious when complete

### Step 2: Define "Done" Criteria

For each task, specify:
- What constitutes completion?
- How to verify it's correct?
- What output/deliverable is produced?

### Step 3: Sequence Tasks

Order by:
- **Dependency:** What must happen first?
- **Risk:** Tackle uncertain parts early
- **Value:** Deliver core value first

### Step 4: Identify Verification Points

Mark where to check quality:
- After each major milestone
- Before irreversible decisions
- When assumptions need validation

## Task Structure Template

```
### Task N: [Clear Task Name]

**Action:** [What specifically to do]

**Details:**
- [Specific detail 1]
- [Specific detail 2]

**Verify:** [How to confirm it's done correctly]

**Output:** [What gets produced]
```

## Examples

### Example 1: Writing Email Plan

```
### Task 1: Draft opening section
**Action:** Write partnership acknowledgment and delay statement

**Details:**
- Reference past collaboration
- State new timeline clearly
- Keep to 2-3 sentences

**Verify:** Opening is warm but direct, timeline is clear

**Output:** Opening paragraph

---

### Task 2: Draft context section
**Action:** Explain integration issue briefly

**Details:**
- One sentence on technical issue
- Non-technical language
- Focus on impact, not details

**Verify:** Technical explanation is < 20 words

**Output:** Context paragraph

---

### Task 3: Draft response section
**Action:** Describe our mitigation actions

**Details:**
- What we're doing to minimize impact
- Timeline for resolution
- Quality commitment

**Verify:** Shows proactive response

**Output:** Response paragraph

---

### Task 4: Draft ask and closing
**Action:** Request extension explicitly

**Details:**
- Clear ask for one-week extension
- Rationale tied to quality
- Confident closing

**Verify:** Ask is unambiguous

**Output:** Final two paragraphs

---

### Task 5: Full review
**Action:** Read entire email aloud

**Details:**
- Check tone consistency
- Verify all points covered
- Polish language

**Verify:** Ready to send

**Output:** Final email
```

**Action:** List causes by impact

- Rank by evidence strength

**Verify:** Top 3 have clear supporting data

## Plan Complexity Levels

### Simple Plan (< 10 tasks)
- Linear sequence
- No dependencies
- Single phase

### Moderate Plan (10-30 tasks)
- Multiple phases
- Some dependencies
- Verification checkpoints

### Complex Plan (30+ tasks)
- Multiple phases
- Complex dependencies
- Frequent verification
- Risk mitigation steps

## Anti-Patterns

### ❌ Vague Tasks

```
Task 1: Research the topic
Task 2: Write the content
Task 3: Review and finalize
```

**Problem:** Not actionable, no verification

### ✅ Specific Tasks

```
Task 1: Find 3 peer-reviewed sources on [topic]
Verify: All sources published after 2020

Task 2: Draft introduction section (200 words)
Verify: Thesis statement is clear

Task 3: Read draft aloud, check for clarity
Verify: No confusing sentences
```

## Handling Dependencies

### Sequential Tasks
```
Task 1 → Task 2 → Task 3
```
Each must complete before next starts.

### Parallel Tasks
```
Task 1 ↘
         → Task 3
Task 2 ↗
```
Tasks 1 and 2 can happen simultaneously.

### Conditional Tasks
```
Task 1 → Decision → Task 2A (if yes)
                  → Task 2B (if no)
```
Next task depends on outcome.

## Verification Checkpoints

**After each major milestone:**
```
### Checkpoint 1: Foundation Complete
- [ ] All research gathered
- [ ] Core structure defined
- [ ] Ready to build

**If any unchecked:** Pause and address before continuing
```

## Transition to EXECUTE

**When to Move On:**
- All tasks defined
- Verification criteria clear
- Sequence logical
- User approves plan (if complex)

**How to Transition:**
```
"Plan complete. Here's the roadmap:

**Total Tasks:** [N]
**Estimated Time:** [X hours/days]
**Verification Points:** [N checkpoints]

Ready to start execution? I'll work through these tasks one by one, showing progress as I go."
```

## Quick Checklist

- [ ] Tasks are 2-10 minutes each
- [ ] Each task has clear "done" criteria
- [ ] Tasks are sequenced logically
- [ ] Verification points identified
- [ ] Plan is approved (if complex)
- [ ] Ready to move to EXECUTE phase

## Remember

**A good plan is:**
- Specific (no ambiguity)
- Verifiable (clear success criteria)
- Sequenced (logical order)
- Realistic (achievable tasks)

**The Rule:** If you can't verify a task is done, it's not well-defined.
