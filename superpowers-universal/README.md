## English

### Overview
A disciplined 5-phase workflow for complex tasks, creative projects, problem-solving, and multi-step work.

**Reference**: This skill is adapted from [obra/superpowers](https://github.com/obra/superpowers/tree/main)

### Core Workflow
```
CLARIFY → DESIGN → PLAN → EXECUTE → VERIFY
```

### The 5 Phases

1. **CLARIFY** - Understand Intent
   - Know WHAT and WHY before deciding HOW
   - Ask clarifying questions (one at a time)
   - Confirm understanding with user

2. **DESIGN** - Create Approach
   - Generate 2-3 alternative approaches
   - Present trade-offs and recommendations
   - Get user approval before proceeding

3. **PLAN** - Break Into Steps
   - Decompose into manageable tasks (2-10 minutes each)
   - Define "done" criteria for each task
   - Sequence tasks by dependency

4. **EXECUTE** - Do The Work
   - Complete one task at a time
   - Show progress regularly
   - Flag issues immediately
   - Adjust plan if needed

5. **VERIFY** - Check Quality
   - Check completeness against plan
   - Verify correctness of outputs
   - Assess quality standards
   - Confirm alignment with original intent

### When to Use

✅ **Always Use For:**
- Complex tasks (> 30 minutes)
- Creative work (writing, design, planning)
- Problem solving with multiple approaches
- Multi-step projects
- Important decisions

⚠️ **Compress But Use For:**
- Moderate tasks (10-30 minutes)
- Time-pressured situations

❌ **Skip Only If:**
- Truly trivial tasks (< 5 minutes)
- Zero ambiguity
- Zero stakes

### Key Principles

1. **No action without clarification** - Understand before acting
2. **Design before planning** - Strategy before tactics
3. **Plan before executing** - Structure prevents drift
4. **Verify before finishing** - Quality requires checking
5. **One question at a time** - Dialogue over monologue

## 中文

### 概述
一个适用于复杂任务、创意项目、问题解决和多步骤工作的结构化5阶段工作流程。

**参考来源**：本技能改编自 [obra/superpowers](https://github.com/obra/superpowers/tree/main)

### 核心工作流
```
澄清 → 设计 → 规划 → 执行 → 验证
```

### 5个阶段

1. **澄清（CLARIFY）** - 理解意图
   - 在决定如何做之前，先明确做什么和为什么
   - 提出澄清性问题（一次一个）
   - 与用户确认理解

2. **设计（DESIGN）** - 创建方案
   - 生成2-3个备选方案
   - 展示权衡和推荐理由
   - 获得用户批准后再继续

3. **规划（PLAN）** - 分解步骤
   - 分解为可管理的任务（每个2-10分钟）
   - 为每个任务定义"完成"标准
   - 按依赖关系排序任务

4. **执行（EXECUTE）** - 完成工作
   - 一次完成一个任务
   - 定期显示进度
   - 立即标记问题
   - 必要时调整计划

5. **验证（VERIFY）** - 检查质量
   - 对照计划检查完整性
   - 验证输出的正确性
   - 评估质量标准
   - 确认与原始意图的一致性

### 何时使用

✅ **始终使用于：**
- 复杂任务（> 30分钟）
- 创意工作（写作、设计、规划）
- 有多种方法的问题解决
- 多步骤项目
- 重要决策

⚠️ **压缩但仍使用于：**
- 中等任务（10-30分钟）
- 时间紧迫的情况

❌ **仅在以下情况跳过：**
- 真正琐碎的任务（< 5分钟）
- 零歧义
- 零风险

### 关键原则

1. **没有澄清就不行动** - 先理解再行动
2. **先设计后规划** - 战略先于战术
3. **先规划后执行** - 结构防止偏离
4. **先验证后完成** - 质量需要检查
5. **一次一个问题** - 对话胜于独白
