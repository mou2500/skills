# Superpowers Universal Skill - 架构总结

## 项目结构

```
superpowers-universal-skill/
├── SKILL.md                          # 主技能入口
├── SUMMARY.md                        # 本文件 - 架构说明
└── references/                       # 子技能/参考文件
    ├── workflow-overview.md          # 完整框架概览
    ├── clarify.md                    # 阶段 1: 澄清
    ├── design.md                     # 阶段 2: 设计
    ├── plan.md                       # 阶段 3: 计划
    ├── execute.md                    # 阶段 4: 执行
    ├── verify.md                     # 阶段 5: 验证
    └── examples.md                   # 完整示例
```

## 核心架构

### 1. 主从技能模式

**主技能 (SKILL.md)**
- 提供技能元数据 (name, description)
- 定义工作流导航逻辑
- 指导何时读取哪个子技能
- 不包含具体执行细节

**子技能 (references/*.md)**
- 每个阶段一个独立文件
- 包含该阶段的详细指导
- 按需加载，渐进式披露
- 可独立更新和维护

### 2. 渐进式加载

```
用户请求
    ↓
读取 SKILL.md (元数据 + 导航)
    ↓
根据当前阶段读取对应子技能
    ↓
执行该阶段
    ↓
阶段完成 → 读取下一子技能
```

### 3. 阶段流转

```
CLARIFY → DESIGN → PLAN → EXECUTE → VERIFY
   ↑                                    ↓
   └────────── (发现问题时返回) ─────────┘
```

## 使用流程

### 场景 1: 新任务开始

```
用户: "帮我写一篇关于 XX 的文章"

AI (读取 SKILL.md):
"我将使用 Superpowers Universal 工作流来完成这个任务。

**当前阶段:** CLARIFY
**目标:** 理解你的需求"

AI (读取 references/clarify.md):
"为了写出符合你需求的文章，我需要先了解一些情况...

首先，这篇文章的目标读者是谁？"

[继续 CLARIFY 流程...]
```

### 场景 2: 用户指定阶段

```
用户: "进入 DESIGN 阶段"

AI (读取 SKILL.md 确认):
"好的，进入 DESIGN 阶段"

AI (读取 references/design.md):
"基于我们刚才澄清的需求，我设计了 3 种方案...

**方案 A:** ...
**方案 B:** ...
**方案 C:** ...

我推荐方案 B，因为..."
```

### 场景 3: 自动流转

```
AI (完成 CLARIFY):
"好的，我已经理解了你的需求。

**总结:** ...

接下来进入 DESIGN 阶段，为你设计具体的方案。"

AI (自动读取 references/design.md):
[展示设计方案]
```

## 优势

### 1. 模块化
- 每个阶段独立维护
- 可以单独更新某个阶段
- 不影响其他阶段

### 2. 渐进式披露
- 不一次性加载所有内容
- 根据当前阶段加载相关指导
- 节省上下文窗口

### 3. 可扩展
- 可以轻松添加新阶段
- 可以为特定领域创建变体
- 可以添加更多示例

### 4. 清晰导航
- 主技能明确指导流程
- 用户清楚当前阶段
- 知道下一步是什么

## 最佳实践

### 对 AI

1. **严格按阶段执行** - 不要跳过阶段
2. **显式声明当前阶段** - 让用户清楚进度
3. **阶段间获取确认** - 特别是 DESIGN → PLAN
4. **展示进度** - 不要静默执行
5. **验证后再完成** - VERIFY 不是可选的

### 对用户

1. **耐心回答澄清问题** - 前期 5 分钟节省后期 30 分钟
2. **认真审阅设计方案** - 这是纠正方向的最后机会
3. **允许 AI 展示进度** - 阶段性检查确保质量
4. **参与验证阶段** - 你的最终确认很重要

## 总结

Superpowers Universal Skill 采用**主从技能架构**：

- **主技能 (SKILL.md)** 负责导航和流程控制
- **子技能 (references/*.md)** 负责具体阶段指导
- **渐进式加载** 确保高效使用上下文
- **清晰流转** 让用户始终知道进度

这种架构使得：
- ✅ 复杂工作流可管理
- ✅ 内容可维护更新
- ✅ 资源使用高效
- ✅ 用户体验清晰
