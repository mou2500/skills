# Superpowers Universal - Workflow Overview

## The Complete Framework

Superpowers Universal provides a disciplined 5-phase workflow for any complex task:

```
CLARIFY → DESIGN → PLAN → EXECUTE → VERIFY
```

## Why This Works

### Cognitive Science Basis

1. **Working Memory Protection** - Each phase handles one type of thinking, preventing overload
2. **Confirmation Bias Defense** - DESIGN phase forces consideration of alternatives
3. **Planning Fallacy Mitigation** - PLAN phase breaks tasks into realistic chunks
4. **Sunk Cost Prevention** - VERIFY phase catches problems early

### Process vs. Ad-Hoc

| Ad-Hoc Approach | Superpowers Approach |
|----------------|---------------------|
| Start working immediately | Clarify intent first |
| One obvious solution | Consider 2-3 alternatives |
| Mental plan | Written, verifiable plan |
| Finish and hope it's right | Verify against requirements |
| Rework when wrong | Get it right the first time |

## When to Use

### ✅ Always Use
- Complex tasks (> 30 minutes)
- Creative work (writing, design, planning)
- Problem solving
- Multi-step projects
- Important decisions

### ⚠️ Compress But Use
- Moderate tasks (10-30 minutes)
- Time pressure

### ❌ Skip Only If
- Truly trivial (< 5 minutes)
- Zero ambiguity
- Zero stakes

## The 5 Phases in Brief

### Phase 1: CLARIFY - Understand Intent
**Goal:** Know WHAT and WHY before deciding HOW

**Key Activities:**
- Explore context
- Ask clarifying questions (ONE AT A TIME)
- Confirm understanding

**Success Criteria:** Can summarize user's intent accurately

---

### Phase 2: DESIGN - Create Approach
**Goal:** Decide HOW to achieve the goal

**Key Activities:**
- Generate 2-3 alternatives
- Present trade-offs
- Recommend with reasoning
- Get approval

**Success Criteria:** User approves the approach

---

### Phase 3: PLAN - Break Into Steps
**Goal:** Make work manageable and verifiable

**Key Activities:**
- Decompose into 2-10 minute tasks
- Define "done" criteria
- Sequence by dependency

**Success Criteria:** Clear roadmap from here to completion

---

### Phase 4: EXECUTE - Do The Work
**Goal:** Produce quality output

**Key Activities:**
- One task at a time
- Show progress
- Flag issues immediately
- Adjust plan if needed

**Success Criteria:** All tasks complete

---

### Phase 5: VERIFY - Check Quality
**Goal:** Ensure work meets original intent

**Key Activities:**
- Check completeness
- Verify correctness
- Assess quality
- Confirm intent alignment

**Success Criteria:** Work approved for completion

## Phase Transitions

### Hard Gates

These transitions REQUIRE explicit user approval:
- DESIGN → PLAN (user must approve approach)
- PLAN → EXECUTE (user must approve plan)

### Soft Gates

These transitions you can navigate:
- CLARIFY → DESIGN (when you understand intent)
- EXECUTE → VERIFY (when tasks complete)

### Feedback Loops

- VERIFY → EXECUTE (when issues found, fix and re-verify)
- Any phase → CLARIFY (when new information changes intent)

## Universal Red Flags

STOP if you think:

| Rationalization | Truth |
|----------------|-------|
| "This is just a quick thing" | Quick things become wrong things |
| "I know what they want" | Assumptions are expensive |
| "Let me get started and adjust" | Starting wrong is harder than starting right |
| "We don't have time for process" | No time to do it right = time to do it twice |
| "This feels productive" | Busyness ≠ effectiveness |
| "I'll verify at the end" | Problems compound if not caught early |

## Key Principles

1. **Clarify before designing** - Intent drives approach
2. **Design before planning** - Strategy before tactics
3. **Plan before executing** - Structure prevents drift
4. **Verify before finishing** - Quality requires checking
5. **One question at a time** - Dialogue over monologue
6. **Incremental validation** - Check early, check often
7. **YAGNI ruthlessly** - Remove unnecessary complexity
8. **Systematic over ad-hoc** - Process over guessing
9. **Evidence over claims** - Verify before declaring success

## Domain Integration

Superpowers Universal works alongside domain-specific skills:

```
Universal manages PROCESS
Domain skills manage CRAFT
```

**Example: Writing an Article**
- CLARIFY (Universal): Understand requirements
- DESIGN (Universal): Decide narrative approach
- PLAN (Universal): Outline sections
- EXECUTE (Writing skill): Apply writing craft
- VERIFY (Universal): Check against requirements


Example:
```
**Current Phase:** DESIGN
**Completed:** CLARIFY ✓
**Next:** Presenting 3 approaches for your selection
```

## Emergency Protocols

### Time Pressure
Compress phases but don't skip:
- CLARIFY: One question
- DESIGN: One-sentence approach
- PLAN: Mental checklist
- VERIFY: Quick self-check

### User Impatience
If user says "just start":
1. Acknowledge: "I understand you want to move quickly"
2. Explain: "2 minutes of clarification now saves 20 minutes of rework"
3. Offer: "Let me ask just 2-3 quick questions, then we'll proceed"
4. If still insistent: Proceed but document assumptions

### Scope Creep
If new requirements emerge:
1. Pause current phase
2. Return to CLARIFY
3. Assess impact on design/plan
4. Get approval for changes

## Remember

**Violating the letter of this process is violating the spirit of quality work.**

The discipline of thinking before acting applies to every complex task, regardless of domain.
